The install command and the exact prompts for claude-seo — the MIT‑licensed skill that turns Claude Code into a parallel SEO audit team. Everything below was verified against the repo on 2026‑07‑22.

The install · copy‑paste into Claude Code
# Inside Claude Code (v1.0.33 or newer) — three lines, run in order:
/plugin marketplace add AgriciDaniel/claude-seo
/plugin install claude-seo@agricidaniel-claude-seo
/seo setup

/seo setup creates an isolated Python environment and installs Playwright Chromium — nothing global touches your machine. Prefer a terminal install instead? git clone --depth 1 https://github.com/AgriciDaniel/claude-seo.git then bash claude-seo/install.sh (Windows: powershell -ExecutionPolicy Bypass -File claude-seo\install.ps1).

The first-audit runbook

1

Verify the install

/seo doctor

You’ll know it worked when doctor reports the Python runtime, dependencies, and Chromium as ready. It changes nothing on your system — it only checks.

2

Run the full audit

/seo audit https://yoursite.com

This is the headline prompt. It crawls up to 500 pages, detects your business type, then fans out up to 15 specialist subagents in parallel — 8 always‑on plus 7 conditional. A typical run takes 10–15 minutes (per the repo’s own benchmark) and consumes Claude usage while it works.

When it finishes you get a 0–100 SEO Health Score plus two files written locally: FULL-AUDIT-REPORT.md and ACTION-PLAN.md — the prioritized fix list, ranked by impact. Nothing is uploaded anywhere; reports stay on your machine.

3

The honest catch

The 0–100 Health Score is Claude’s structured judgment — a model heuristic, not Google ranking data. Treat it as a consistent yardstick between runs, not an oracle.

Without Google API credentials, Core Web Vitals are lab estimates only; live rank, volume, and backlink data needs opt‑in extensions (DataForSEO, Ahrefs) with their own accounts.
