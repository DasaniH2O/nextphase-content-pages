Setup Guide · Claude Code × Obsidian

Point Claude Code at your vault and it links your notes while you sleep.

An Obsidian vault is just a folder of plain-text .md files. Claude Code reads and writes local files from the terminal. Put a scheduled job between them and every night it crawls your newest notes, wiki-links them to older ones, flags contradictions, and drafts a “what your brain missed” note. No plugin — because it’s all markdown.

The nightly pipeline

One command a night. Claude Code opens the vault, reads the map you generated once, and touches only markdown.

Set it up once

1

Install Claude Code and open the vault folder

Install the CLI (npm i -g @anthropic-ai/claude-code), then cd into the folder your .obsidian settings live in — that folder is your vault. You’ll know it’s right when ls -a shows a .obsidian directory.

2

Run /init to generate the map

Inside Claude Code, type /init. It writes a CLAUDE.md at the vault root describing your folder structure, note conventions, and tags. This is the file the nightly job reads first — edit it once to tell the agent your linking rules.

3

Save the nightly instruction as a prompt file

Drop a plain night-shift.md in the vault with the job: “Read notes modified in the last 24h. Add [[wikilinks]] to related older notes. Flag any note that contradicts another. Write one dated Brain-Missed.md summary.” Exact instructions beat clever ones here.

4

Schedule it with cron

A cron job is a task your machine runs on a clock. Add one line so it fires at 2 a.m. — assuming the machine is on and awake at that hour.

# crontab -e  → add one line:
0 2 * * *  cd ~/vault && claude -p "$(cat night-shift.md)" \
           --allowedTools "Read,Edit,Write" >> ~/vault/.night.log 2>&1
5

Verify the first run before you trust it

Next morning, open .night.log and the new Brain-Missed.md. Run git diff to see exactly which notes it edited. You’ll know it worked when the diff is small, sane, and only touches markdown. If it rewrote whole notes, tighten the prompt in step 3 and narrow --allowedTools.

Prerequisites

  • Obsidian — free; the vault is a plain folder of .md files
  • Claude Code — needs a Claude Pro/Max plan or API credit; burns usage every run
  • Terminal + cron — built into macOS/Linux; Task Scheduler on Windows
  • git — strongly advised before step 4

Cost & time

  • Setup: ~20 min, once
  • Per run: a few minutes of Claude Code usage; larger vaults (thousands of notes) run slower and cost more
  • No plugin, no subscription beyond Claude Code

The honest catch

This job needs read and write access to your entire second brain, unattended. Before you trust it: put the vault under git, commit a clean baseline, and back it up. Then every night’s changes are a reviewable, revertible diff — not a silent overwrite.
