GitHub Spec Kit · setup

Turn “build me an app and pray” into an executable spec — the exact Spec Kit setup

Spec Kit (123k stars) makes you write the spec first, then drives your AI agent through a fixed sequence so the spec generates the build. Here is the precise install and command order — and the honest prerequisites nobody mentions.

Prerequisites (get these first)

  • uv (the Python package manager)
  • Python 3.11+
  • Git
  • A supported AI coding agent — 30+ work (Claude Code, Cursor, Codex, …)

Install & init

# install the CLI
uv tool install specify-cli
# scaffold your project for a specific agent
specify init my-project --integration claude

Then run the sequence in your agent

1
/speckit.constitution — set the project’s non-negotiable rules first.
2
/speckit.specify — describe what and why, not the stack.
3
/speckit.plan — the agent proposes the technical plan.
4
/speckit.tasks — break the plan into ordered, checkable tasks.
5
/speckit.implement — the agent builds against the spec, task by task.
7 core commands + 3 optional (10 total). The spec is the source of truth — when the build drifts, you fix the spec, not the code.
