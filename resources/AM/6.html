Install Hallmark once and your AI-built UIs stop looking AI-built

Hallmark is a free, MIT-licensed design skill — a rule-set your agent reads, not an app you run. It forces the agent off the default layout every model reaches for, dresses the page in one of twenty themes, then runs the output through a 57-gate slop test before it hands you anything.

Install — one command

1

Add the skill to your agent

Works in Claude Code, Cursor, or Codex. Run this in your project (or globally) and the skill file installs itself.

npx skills add nutlope/hallmark

Manual alternative: copy SKILL.md + references/ into ~/.claude/skills/hallmark/, .cursor/rules/hallmark.mdc, or ~/.codex/skills/hallmark/.

2

Confirm the agent picked it up

Start a new session and ask for a UI. You’ll know it worked when the agent narrates a macrostructure choice and a named theme instead of jumping straight to a centered hero and three icon cards.

3

Call a verb

Default behaviour is build. The other three are named below — type the verb plus a target.

The four verbs — centerpiece

What each verb does · as of 2026-07-20
VerbWhat it doesTouches your copy?
buildDefault. Generates a new UI off a non-default page structure + one of 20 themes.Writes new
auditScores an existing page against the 57 gates. Reports only — makes no edits.No
redesignRebuilds a page’s look while keeping its copy, information architecture, and brand.Keeps yours
study <screenshot|URL>Extracts the “design DNA” of a reference so you can reuse the vibe, not the pixels.No

One honest limitationHallmark only shapes those three agents — it is a rule-set, not a standalone app, so it does nothing in Figma, v0, or a raw ChatGPT window. And study deliberately refuses to pixel-clone a site or copy a paid template; it pulls principles, not a 1:1 rip.

What “slop” means
The generic AI look: centered everything, three icon cards, purple-blue gradients, rounded-2xl + soft shadow on every box. The gates flag these before you see the output.
