Prompt caching · API cost

Cache the front of your prompt — pay one-tenth for the part that never changes

Your tools and system instructions are byte-identical on every call. Mark them once and the provider reads back the computed version at a steep discount. Here is exactly where the breakpoint goes, for Anthropic and OpenAI.

The one move

1
Put the static stuff first. Order the request tools → system → long context → the turn-by-turn messages. Anything that changes every call (the user’s latest message, a timestamp) goes last.
2
Set the breakpoint on the last static block. On Anthropic that’s an explicit marker; everything before it gets cached. On OpenAI it’s automatic for any prompt over ~1,024 tokens.
3
Keep that prefix byte-for-byte identical. This is the whole game. One changed character before the breakpoint and you pay full price again (see the catch).

The exact setup

Anthropic — Messages API 
# mark the END of the static prefix. tools + system + everything
# above this point is cached and read back at 0.1x next time.
{
  "model": "claude-opus-4-8",
  "system": [
    { "type": "text", "text": "<your long, fixed system prompt>",
      "cache_control": { "type": "ephemeral" } }   ← breakpoint
  ],
  "messages": [ { "role": "user", "content": "<the part that changes>" } ]
}
OpenAI — automatic 
# nothing to set. just keep the static content at the FRONT
# and identical. caching kicks in above ~1,024 tokens,

Break-even: a cache write costs more than a normal call first — 1.25x for the 5-minute cache, 2x for the 1-hour. So you need at least one hit (5-min) or two hits (1-hour) on that prefix before it’s cheaper than not caching. Agents that loop on the same system prompt clear this instantly; a one-off call never does.

OpenAI

Automatic. 50% off the cached prefix, prompts >1,024 tokens, in 128-token steps. Cache clears after 5–10 min idle.

The catch — what breaks it

Any of these before the breakpoint invalidates the whole cache:
  • a timestamp or per-request variable up front
  • changed tool definitions
  • toggling web search or citations
  • adding or removing an image
  • the 5-min TTL lapsing on bursty traffic

Straight talk

The “70% cheaper” you saw is workload-dependent, not a flat rate. The real read discounts are 90% (Anthropic) and 50% (OpenAI) — on the reused portion only, after you clear the write premium.
